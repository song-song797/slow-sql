    2.  **SQL 语法适配性**：
        -   语法符合 MySQL 规范。
        -   使用 `SELECT *`：在字段较少且单行返回场景下，性能损耗可忽略；若表包含大文本字段（如 TEXT/BLOB），即使单行也可能增加少量 IO，但鉴于 `cust_id` 通常为整型主键或唯一键，此风险极低。
    3.  **索引使用预判**：
        -   查询条件为 `cust_id = 20001` (等值查询)。
        -   **场景 A（理想）**：若 `cust_id` 是主键（Primary Key）或拥有唯一索引（Unique Key），MySQL 将直接通过 B+ 树定位到唯一行，时间复杂度 O(1)，性能极佳。

    4.  **性能风险点**：
        -   当前表行数显示为 0 或极少，属于小表查询。
        -   即使是全表扫描，数据量极小，IO 耗时可忽略不计。
